## GB Block Output Pattern — Section Wrapper Structure

**Date:** 2026-06-05
**Source:** User's existing GB blocks (MINO Digital site)

### The Pattern

Every section follows a two-container wrapper system:

```
<!-- Outer wrapper: controls section-level properties -->
<div class="section-outer__margin__LR section-outer__radius section-outer__margin-bot bgc-dark">
  <!-- Inner container: controls content width + padding -->
  <div class="section-inner__L">
    <!-- actual section content -->
  </div>
</div>
```

### Outer Wrapper (section-outer)

| Global Class | Purpose |
|---|---|
| `.section-outer__margin__LR` | Left/right margin (`var(--padding-m)`, 0 on mobile) |
| `.section-outer__radius` | Border-radius (`1rem`, 0 on mobile) |
| `.section-outer__margin-bot` | Bottom margin (`var(--padding-s)`, 0 on mobile) |
| `.bgc-dark` / `.bgc-shaded` | Background color variant |

### Inner Container (section-inner)

| Global Class | Properties |
|---|---|
| `.section-inner__L` | `max-width: var(--gb-container-width)`, `margin: 0 auto`, `padding: var(--padding-xl)` |
| `.section-inner__L__bot-half-padding` | Same but `padding-top/bottom: var(--padding-m)` |
| `.section-inner__L__top-half-padding` | Same but half padding variant |
| `.section-inner__M` | `max-width: var(--gb-container-width)`, `padding: var(--padding-l)` |

The inner container's `max-width` uses `--gb-container-width`, which is generated by GB from the **GP settings `container_width`** (1264px on this site).

### Spacing Variables

| Variable | Value | Context |
|---|---|---|
| `--gb-container-width` | 1264px | From GP `container_width` |
| `--padding-xl` | ~120px | Largest spacing |
| `--padding-l` | ~80px | Large spacing |
| `--padding-m` | ~40px | Medium spacing |
| `--padding-s` | ~20px | Small spacing |
| `--padding-xs` | ~10px | Extra small spacing |

### Implication for Converter

1. **`<main>` must be stripped** — Done. WordPress/GP theme handles the document structure. Top-level blocks should be the section wrappers directly.

2. **Section detection & pattern application** — Source HTML sections (originally using Tailwind for layout) should be mapped to this outer/inner pattern:
   - Source `<section class="pt-32 pb-20 px-6 max-w-[1280px] mx-auto">` → becomes two blocks: outer wrapper with margin/radius classes + inner container with `section-inner__L`
   - The `container_width` GP setting must match the source design's max-width

3. **`container_width` extraction is critical** — The Tailwind config's max-width or the largest `max-w-*` class on container elements should become the GP `container_width`. This syncs `--gb-container-width` with the design.

4. **For the global styles pipeline** — These section-outer/section-inner classes are prime candidates for pre-registered global styles. A standard page has ~10 sections, each reusing the same 3-4 outer classes. No need to resolve these to inline styles — just ensure the Global Style CSS exists in the DB.

5. **For the converter** — Currently the walker produces flat sections. Future: detect if a section's outermost `<div>` child is a content-width container, and wrap it in the outer/inner pattern with appropriate global class references.
