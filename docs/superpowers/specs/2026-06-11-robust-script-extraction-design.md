# Robust Script Extraction — Design

**Date:** 2026-06-11
**Status:** Approved

## Problem

The converter strips all `<script>` tags from source HTML and outputs them in
`global.js`. External CDN scripts become comments, inline scripts are preserved
as-is but concatenated into a single block. One inline script that references a
missing global (e.g., `tailwind.config` when the Tailwind CDN isn't enqueued)
throws a `ReferenceError` that kills all subsequent scripts — Swiper, GSAP,
Lucide, everything.

This is systemic: every project with `<script>` tags is affected. The user has
no way to know which CDN scripts are needed or which inline scripts depend on
them.

## Goal

Make `global.js` robust against cascading failures, and make dependencies
self-documenting so users know exactly what to enqueue before loading it.

## Architecture

Three changes to `src/core/script-extractor.ts`. No new files. No changes to
other modules.

```
extractScripts()          (unchanged — extracts raw scripts)
       ↓
deduplicateScripts()      (unchanged — deduplicates by src/content)
       ↓
[detectDependencies()]    (NEW — scans inline scripts for global refs)
       ↓
formatGlobalJs()          (MODIFIED — wraps each block in IIFE+try-catch,
                           annotates with dependency comments, no longer
                           concatenates all inline scripts into one blob)
       ↓
manual-steps generator    (MODIFIED in orchestrator or CLI — reads
                           dependencies and adds CDN checklist section)
```

## Components

### Change 1: Error isolation (`formatGlobalJs`)

Each inline script gets wrapped:

```js
// -- From index.html (references: tailwind)
(function() {
  try {
    tailwind.config = { ... };
  } catch(e) {
    console.warn('[gb-converter] Script from index.html failed:', e.message);
  }
})();
```

Each inline block is a separate IIFE. A failure in one does not affect others.
`console.warn` is used (not `console.error`) so it doesn't pollute error
tracking — the user can still open devtools to see which scripts failed.

External scripts remain as comments (unchanged).

### Change 2: Dependency detection (`detectDependencies`)

New function that scans each inline script's content for known global references.
A curated set of known library globals is used (not arbitrary regex — this is
intentional to avoid false positives on local variable names):

```ts
const KNOWN_GLOBALS: Record<string, string> = {
  "tailwind": "Tailwind CDN",
  "gsap": "GSAP",
  "ScrollTrigger": "GSAP ScrollTrigger",
  "Swiper": "Swiper",
  "lucide": "Lucide Icons",
  "createIcons": "Lucide Icons",
  // Extensible as new libraries are encountered
};
```

Detection scans for `\bglobalName\b` in the script content. Each match is
recorded and associated with the script's source page.

The annotation appears in the inline scripts section header and as per-script
comments:

```js
// === Inline Scripts ===
// ⚠ These scripts require CDN libraries. See "External Scripts" above
//    for wp_enqueue_script() calls. Scripts will silently skip if their
//    dependencies are not loaded.

// -- From index.html (requires: tailwind)
(function() { ... })();

// -- From index.html (requires: Swiper)
(function() { ... })();
```

### Change 3: `manual-steps.txt` CDN checklist

The CLI (or orchestrator) reads script dependencies and appends a section to
`manual-steps.txt`:

```
## Required CDN Scripts

These <script> tags were stripped from your source HTML during conversion.
Enqueue them before loading global.js:

  <script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/gsap.min.js"></script>
    → Used by: GSAP animations (index.html)
  <script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/ScrollTrigger.min.js"></script>
    → Used by: GSAP animations (index.html)
  <script src="https://unpkg.com/lucide@latest"></script>
    → Used by: lucide icons (index.html)
  <script src="https://cdn.jsdelivr.net/npm/swiper@11/swiper-bundle.min.js"></script>
    → Used by: Swiper carousel (index.html)

Add them as a WPCodeBox HTML snippet with location set to "Header".
```

This is generated by the CLI after `global.js` is written — the script entries
(including external ones) are already available.

## Data Flow

```
Source HTML:
  <script src="gsap.min.js"></script>
  <script>gsap.to(...)</script>
  <script>tailwind.config = {...}</script>
  <script>new Swiper(...)</script>
       │
       ▼ extractScripts()
  [
    { type: "external", src: "gsap.min.js" },
    { type: "inline", content: "gsap.to(...)" },
    { type: "inline", content: "tailwind.config = {...}" },
    { type: "inline", content: "new Swiper(...)" },
  ]
       │
       ▼ deduplicateScripts()  [unchanged]
  [...deduplicated entries...]
       │
       ▼ detectDependencies(inlineScripts)
  [
    { content: "gsap.to(...)", globals: ["gsap"] },
    { content: "tailwind.config = {...}", globals: ["tailwind"] },
    { content: "new Swiper(...)", globals: ["Swiper"] },
  ]
       │
       ▼ formatGlobalJs(entries, dependencies)
  global.js with IIFE wrapping + dependency annotations
       │
       ▼ CLI: generateManualSteps(entries, dependencies)
  manual-steps.txt with CDN script checklist
```

## Error Handling

- **Empty inline script:** No wrapping needed, skip
- **No dependencies detected:** Still wrap in IIFE+try-catch for consistency
- **External script URL is malformed:** Already handled by `slugFromUrl` with
  fallback sanitization
- **`KNOWN_GLOBALS` misses a library:** The script still runs (wrapped in
  try-catch), but the dependency annotation is missing. No crash. Add the
  missing global to the map when discovered.

## Testing

### Modified: `tests/script-extractor.test.ts`

New test cases:

- Each inline script wrapped in IIFE with try-catch
- Error in first script does not prevent second script from executing
- Dependency annotation appears in output for known globals
- Script with no known globals still gets IIFE wrapping
- Scripts with multiple dependencies get all listed
- Empty scripts produce no wrapping
- External-only scripts produce no inline section
- `detectDependencies` correctly identifies `gsap`, `Swiper`, `lucide`, `tailwind`, `ScrollTrigger`

Existing tests (deduplication, extraction, formatting) should still pass
with the updated `formatGlobalJs` signature.

## What This Does NOT Change

- Extraction logic (regex-based script extraction unchanged)
- Deduplication logic
- External script comment format
- Preprocessor, DOM walker, serializer, CSS splitter — zero changes
- Existing `global.js` structure (external section + inline section preserved)

## Future Extension

The `KNOWN_GLOBALS` map is the single extension point. When a new project
uses a library not in the map, add it:

```ts
"bootstrap": "Bootstrap JS",
"alpine": "Alpine.js",
"lenis": "Lenis Smooth Scroll",
```
