Please make the next patch a robustness patch, not a visual-design patch.

Goal: centered single-column heroes should normalize and score correctly without weakening existing two-column hero detection.

Scope:

Update hero-intake.ts

Replace the current broad forced-grid heuristic that triggers whenever the inner wrapper has display:flex, display:grid, or max-width. Right now that logic flattens constrained centered heroes into a fake two-column section grid, which is causing bad normalization.

Introduce three structural paths:

real multi-column hero wrapper,

centered constrained single-column wrapper,

fallback generic parsing.

Preserve source-derived structure and styleIntent; do not synthesize a full visual design during normalization.

For centered heroes, keep the IR as:

section

child constrained container

ordered content children such as eyebrow / heading / paragraph / CTA row / stats row.

Add a centered-children parser that is structure-first:

preserve headings, paragraphs, links, images in order,

group repeated stat items into one stats-row container when the repeated child shape strongly suggests metric + label,

group multiple adjacent CTA links/buttons into one CTA row,

avoid substring-only heuristics like includes("stat") unless combined with child-shape checks.

Update hero-scorer.ts

Keep the existing scoring API, but add:

hasStatsRow

hasCenteredStack

Narrow hasTwoColumns so a CTA row or stats row cannot satisfy it. A row container by itself is not enough to mean a two-column hero.

Detect a real two-column hero from structural split/grid behavior at the hero-content level, not from any local row container.

Add family-aware scoring:

Family A = two-column hero,

Family B = centered constrained hero.

Make sure:

existing two-column hero fixtures still pass strongly,

centered hero with heading + paragraph/CTA + stats can clear the pattern threshold,

generic constrained single-column sections do not get promoted too easily.

Do not change hero-converter.ts

Leave converter orchestration unchanged unless strictly required.

Add fixtures before tuning threshold

Add a centered hero fixture with constrained centered layout, heading, paragraph, CTA group, and stats row.

Add at least one non-hero row/grid fixture to prove CTA rows or stats rows do not accidentally count as hasTwoColumns.

Keep existing two-column hero behavior passing.

Deliverables:

updated hero-intake.ts

updated hero-scorer.ts

new/updated fixtures

short note explaining how hasTwoColumns, hasStatsRow, and hasCenteredStack are now distinguished.
